Part 2 — Iteration 1: Initial Scoring

Code Audit Results

A1 — MCP Tool Surface

Platform Architect — 3/10
Finding: README says "35 tools available." Actual count from mcp.js: 45 unique tools. The tools section lists 35 but omits 10 tools added since 1.1.5: figma_batch, figma_create_ellipse, figma_create_chart, figma_discover_library_styles, figma_discover_library_variables, figma_preload_variables, figma_restyle_artboard, figma_tag_raw_exception, figma_validate_ds_compliance, mimic_render_url.
Assertion: Public claim is 22% understated. A user evaluating the tool surface sees fewer capabilities than exist.

A16 — Privacy

Platform Architect — 10/10
Finding: Audited mcp.js and bridge.js. Only outbound calls: (1) Figma REST API (api.figma.com) for published component keys and file metadata, (2) localhost bridge (127.0.0.1:3055). No telemetry, no analytics, no external logging. Puppeteer runs locally for rendering.
Assertion: Claim is accurate. Design data never leaves the machine.

A17 — Install Path

Platform Architect — 9/10
Finding: install.sh exists and handles clone + env setup. npx -y @miapre/mimic-ai works (confirmed: npm registry has 1.3.0). One gap: README says npx mode "doesn't set FIGMA_ACCESS_TOKEN" but doesn't explain what happens — user gets a confusing error on first build.
Assertion: Install path works. Minor UX gap in npx error messaging.

A18 — Rule Count

Product QA — 10/10
Finding: GOLDEN_RULES.md has 46 section headings (## 1 through ## 46). README says "46 quality rules." Match confirmed.
Assertion: Claim is accurate.

A19 — Strict Mode

DS Integration Engineer — 9/10
Finding: set_session_defaults accepts dsMode: "strict". In strict mode, create_frame, create_text, set_node_fill, and set_layout_sizing all check for raw values and throw DS_STRICT_VIOLATION. The 1.3.0 fix for set_layout_sizing now accepts DS variable path strings in strict mode. validate_ds_compliance post-build audit exists.
Assertion: Strict mode works as advertised. -1 because strict mode enforcement is in the orchestrator's protocol (CLAUDE.md), not structurally enforced in every tool handler — a new MCP client could bypass it.

MCP Tool Invocation Results

A8 — Build Reports

Learning Engineer — 10/10
Finding: mimic_generate_build_report tested with full params (screenName, dsName, toolCallCount:67, cacheHits:22, coldBuildEstimate:140, primitives with instances, format:html). Output contains: What was built, DS usage, Quality table, Patterns table with cache status, Efficiency section, DS gap recommendations with savings, Conventions learned. HTML tables render cleanly (line-by-line parser fix confirmed).
Assertion: Build report generation works as advertised.

A9 — Efficiency Tracking

Learning Engineer — 10/10
Finding: Report includes Efficiency section with table: Tool calls (67), From cache (22), Estimated cold build (~140), Saved vs cold (~73 calls). Summary object returns toolCalls: 67, cacheHits: 22, savedVsCold: 73. Component savings projection present: "If your DS added these as components, future builds would save ~90 calls/build."
Assertion: Efficiency tracking works exactly as documented in CHANGELOG 1.3.0.

A10 — DESIGN.md Generation

Learning Engineer — 10/10
Finding: mimic_generate_design_md tested. Output: 285 colors (not 570 — dedup confirmed), 74 text styles, 20 spacing tokens, 11 radius tokens, 16 components with pattern mappings. YAML frontmatter is machine-readable. Spacing shows token for unresolvable values (not null). No fake fontSize entries.
Assertion: DESIGN.md generation works as advertised. All 1.3.0 fixes confirmed.

A20 — Gap Recommendations with Savings

Learning Engineer — 10/10
Finding: Gap entries include savings: "card/metric — With a DS component: ~20 fewer calls/build." Cross-references primitives array instances field for per-gap projections. Gaps without instance data get generic "Adding this to your DS would reduce tool calls."
Assertion: Gap recommendations include savings estimates as promised.
